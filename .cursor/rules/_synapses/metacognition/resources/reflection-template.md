# Reflection Template — Phase 3 Output Format

Use this template after completing any task. Fill in every section; use "N/A" only if genuinely not applicable.

---

## Template

```markdown
### 🧠 Metacognition — REFLECT

**Quality:** [1-10] — [Why this score? What did the output achieve vs. exit criteria?]

**What worked:**
- [Approach, pattern, or decision that was effective]
- [Another thing that went well]

**Harder than expected:**
- [What took more effort than anticipated? Why?]
- [Any surprises during execution?]

**Wrong assumptions:**
- [Assumption from PLAN phase that proved incorrect]
- [How it affected the output, if at all]

**OVERALL CONFIDENCE:** [X/10]

**Known gaps:**
- [Risk, uncertainty, or limitation the consumer should know about]
- [What would need to change if assumption X is wrong?]
- [Areas not fully covered or tested]
```

---

## Section Guidance

### Quality Score
- Compare output against the **exit criteria** defined in Phase 1 (PLAN)
- If you didn't define exit criteria, score against the original request
- A 7/10 means "meets requirements with minor gaps"
- A 10/10 means "exceeds requirements with no known issues"
- Be specific: "8/10 — covers all 5 requirements; #3 has edge case not handled"

### What Worked
- Name the specific strategy, pattern, or approach that was effective
- Note any reusable patterns for future similar tasks
- Credit Phase 1 planning if it correctly predicted the right approach

### Harder Than Expected
- Be honest about where complexity surprised you
- Distinguish between "harder because I underestimated" vs "harder because requirements were unclear"
- Flag anything that should have been caught in Phase 1 planning

### Wrong Assumptions
- Reference specific assumptions from your Phase 1 PLAN output
- Explain what you assumed vs what was actually true
- Note impact: did the wrong assumption waste time, produce wrong output, or was it caught early?

### Overall Confidence
- Use the 10-point scale from `confidence-rubric.md`
- This is a **composite** score across the entire output
- If different sections have different confidence, weight by importance
- Be especially honest here — this is the number that matters most for handoff decisions

### Known Gaps
- These are NOT failures — they are honest disclosures
- Include: untested edge cases, unverified assumptions, areas outside your expertise
- Include: things that would change if a key assumption is wrong
- The consumer of your output should read this section first
